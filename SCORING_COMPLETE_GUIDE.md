## 📊 점수 체계 개요

### 3가지 점수 카테고리
1. **성경 점수** (Scripture Score) - 노란색 🟡
2. **기도 점수** (Prayer Score) - 빨간색 🔴  
3. **활동 점수** (Activity Score) - 초록색 🟢

### 종합 점수 (Ministry Score)
```
종합 점수 = 성경 점수 + 기도 점수 + 활동 점수
```

## 👍 2. 반응 버튼 점수 (일반 포스팅)

### A. 말씀 포스팅 반응 (📖 "아멘!")

#### 클릭 시
- **반응 클릭자**: 성경 점수 **+1점**
- **포스트 원저자**: 성경 점수 **+2점** (본인 글 제외)

#### 취소 시
- **반응 클릭자**: 성경 점수 **-1점** (0점 이하 방지)
- **포스트 원저자**: 성경 점수 **-2점** (본인 글 제외, 0점 이하 방지)

### B. 일상/사역/찬양/교회/자유 포스팅 반응

#### 클릭 시
- **반응 클릭자**: 활동 점수 **+1점**
- **포스트 원저자**: 활동 점수 **+2점** (본인 글 제외)

#### 취소 시
- **반응 클릭자**: 활동 점수 **-1점** (0점 이하 방지)
- **포스트 원저자**: 활동 점수 **-2점** (본인 글 제외, 0점 이하 방지)

## 🎯 점수 계산 예시

### 예시 1: 일반 사용자 하루 활동
```
[아침]
- 성경 구절 타이핑 완료: 성경 점수 +100점
- 설교 영상 시청 완료: 성경 점수 +100점

[낮]
- 중보 포스팅 작성: 기도 점수 +10점
- 말씀 포스팅 작성: 성경 점수 +10점
- 일상 포스팅 작성: 활동 점수 +10점

[저녁]
- 친구 중보 글에 기도하기: 기도 점수 +20점
- 친구 말씀 글에 반응: 성경 점수 +1점
- 친구 일상 글에 댓글: 활동 점수 +5점

총점:
- 성경 점수: 100 + 100 + 10 + 1 = 211점
- 기도 점수: 10 + 20 = 30점
- 활동 점수: 10 + 5 = 15점
- 종합 점수: 256점
```

### 예시 2: 인기 게시글 작성자
```
[작성자가 받는 점수]
- 말씀 포스팅 작성: 성경 점수 +10점
- 10명이 반응 클릭: 성경 점수 +20점 (2점 × 10명)
- 5명이 댓글 작성: 활동 점수 +25점 (5점 × 5명)

작성자 총점:
- 성경 점수: 10 + 20 = 30점
- 활동 점수: 25점
- 종합 점수: 55점

[반응/댓글 작성자들이 받는 점수]
- 반응 클릭자 각각: 성경 점수 +1점
- 댓글 작성자 각각: 활동 점수 +5점
```

## 🔄 점수 반영 방식

### 실시간 반영 ✅
- 포스팅 작성/삭제 즉시 점수 업데이트
- 반응 버튼 클릭/취소 즉시 점수 업데이트
- 댓글 작성/삭제 즉시 점수 업데이트
- 타이핑/비디오 완료 즉시 점수 업데이트
- API 응답에서 업데이트된 점수를 받아 UI 동기화

### 점수 차감 규칙
- **모든 점수는 0점 이하로 내려가지 않음** (`Math.max(0, score - deduction)`)
- 본인 글에 본인이 반응/댓글 시 원저자 보너스 없음
- 삭제 시 획득했던 점수만 차감

## 💾 데이터베이스 스키마

### users 테이블 점수 관련 필드
```sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  email TEXT UNIQUE NOT NULL,
  name TEXT NOT NULL,
  
  -- 점수 필드
  scripture_score INTEGER DEFAULT 0,     -- 성경 점수
  prayer_score INTEGER DEFAULT 0,        -- 기도 점수
  activity_score INTEGER DEFAULT 0,      -- 활동 점수
  
  -- 완료 기록 (JSON 배열)
  completed_verses TEXT DEFAULT '[]',    -- 완료한 성경 구절 날짜 목록
  completed_videos TEXT DEFAULT '[]',    -- 완료한 설교 영상 ID 목록
  
  -- 기타 필드...
)
```

### 관련 테이블
```sql
-- 일반 포스팅 반응
CREATE TABLE likes (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  post_id INTEGER NOT NULL,
  user_id INTEGER NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(post_id, user_id)
)

-- 기도 포스팅 반응 (별도 관리)
CREATE TABLE prayer_clicks (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  post_id INTEGER NOT NULL,
  user_id INTEGER NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(post_id, user_id)
)

-- 댓글
CREATE TABLE comments (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  post_id INTEGER NOT NULL,
  user_id INTEGER NOT NULL,
  content TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
)
```

## 📍 주요 API 엔드포인트

### 점수 조회
```
GET /api/users/:id/scores
Response: {
  scripture_score: 211,
  prayer_score: 30,
  activity_score: 15,
  total_score: 256,
  completed_videos: ["video_20260219"],
  completed_verses: ["2026-02-19"]
}
```

### 포스팅 작성 (자동 점수 업데이트)
```
POST /api/posts
Body: { user_id, content, background_color, ... }
Response: { 
  id, 
  updated_scores: { 
    scripture_score?: 110,
    prayer_score?: 20,
    activity_score?: 20
  }
}
```

### 반응 버튼 (일반 포스팅)
```
POST /api/posts/:id/like
Body: { user_id }
Response: { 
  liked: true, 
  scripture_score?: 212,  // 말씀 포스팅
  activity_score?: 16     // 일상/사역/찬양/교회/자유 포스팅
}
```

### 기도하기 버튼 (중보 포스팅)
```
POST /api/posts/:id/pray
Body: { user_id }
Response: { 
  prayed: true, 
  prayer_score: 50  // +20점
}
```

### 댓글 작성 (자동 점수 업데이트)
```
POST /api/posts/:id/comments
Body: { user_id, content }
Response: { 
  id, 
  new_activity_score: 20  // 댓글 작성자의 새로운 활동 점수
}
```

```

}

```

### 비디오 완료
```
POST /api/users/:id/scores/video
Body: { video_id: "video_20260219" }
Response: { 
  success: true,
```

## 📌 중요 참고 사항

1. **종합 점수는 별도 저장하지 않음**
   - 프론트엔드에서 실시간 계산: `scripture_score + prayer_score + activity_score`
   - 관리자 패널에서도 동일하게 계산

2. **본인 글 제외 로직**

3. **0점 이하 방지**
   - 모든 점수 차감 시 `Math.max(0, currentScore - deduction)` 사용
